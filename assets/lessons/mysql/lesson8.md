# Lesson 8: "Key Strategies: Surrogate vs. Natural & Auto-Increment"

မိူဝ်ႈႁဝ်းလိူၵ်ႈ Attribute ၼိုင်ႈဢၼ်မႃးႁဵတ်းပဵၼ် **Primary Key (PK)** ၼၼ်ႉ ႁဝ်းလူဝ်ႇမီးလၢႆးဝူၼ်ႉ (Strategy) ၶႃႈ။ 
ၵႆႉမီးလွင်ႈထၢမ်ဝႃႈ "တေဢဝ်ၶေႃႈမုၼ်း ဢၼ်မီးယူႇယဝ်ႉမႃးၸႂ်ႉႁႃႉ ဢမ်ႇၼၼ် တေတႅမ်ႈမၢႆၼပ်ႉမႂ်ႇမႃးၸႂ်ႉ?" ၼႆၶႃႈ။

### 1. ပိူင်သၢင်ႈ (The Concept)
ဝူၼ်ႉတူၺ်း "ပပ်ႉလိၵ်ႈ" ၼႂ်းႁွင်ႈတူၺ်းလိၵ်ႈ (Library) ၼိုင်ႈပပ်ႉၶႃႈ...
* **Natural Key:** ၸႂ်ႉၶေႃႈမုၼ်းဢၼ်မီးယူႇၸွမ်းတူဝ်မၼ်း - မိူၼ်ၼင်ႇ မၢႆ **ISBN** ဢၼ်တူၵ်းမႃးၸွမ်းပပ်ႉ။
* **Surrogate Key:** သၢင်ႈမၢႆတွင်းမႂ်ႇၶိုၼ်း တွၼ်ႈတႃႇ Database ႁဝ်းၵူၺ်း - မိူၼ်ၼင်ႇ **Book_ID (1, 2, 3...)**။



### 2. Natural Key (သေႃးသၽႃႇဝ)
ပဵၼ်ၵၢၼ်ဢဝ် Attribute ဢၼ်မီးဝႆႉ ၼႂ်းလူၵ်ႈလေႃးၵ မႃးႁဵတ်းပဵၼ် Primary Key ၶႃႈ။
* **Examples:** NRC ID (မၢႆၾၢင်), Passport Number, Product Serial Number။
* **Advantage:** ဢမ်ႇလူဝ်ႇသၢင်ႈ Column မႂ်ႇ၊ ၶေႃႈမုၼ်းမီးတီႈပွင်ႇၼႂ်းတူဝ်မၼ်း။
* **Disadvantage:** သင်ၶေႃႈမုၼ်းၼၼ်ႉ လႅၵ်ႈလၢႆႈ (မိူၼ်ၼင်ႇ ပိူၼ်ႈလႅၵ်ႈ Passport မႂ်ႇ)၊ မၼ်းတေယုင်ႈတေႃႇ Database တင်းသဵင်ႈၶႃႈ။

### 3. Surrogate Key (သေႃးသၢင်ႈမႂ်ႇ)
ပဵၼ်ၵၢၼ်သၢင်ႈ Column မႂ်ႇဢၼ်ၼိုင်ႈ ဢၼ်ဢမ်ႇမီးတီႈပွင်ႇသင် ၼႂ်းၵၢၼ်ပဵၼ်ယူႇၵူႈဝၼ်း ၵူၺ်းၵႃႈ ၸႂ်ႉတႃႇမၢႆတွင်း Row ၼႂ်း Database ၵူၺ်းၶႃႈ။
* **Examples:** `user_id`, `order_id`။
* **Advantage:** မၼ်းတေဢမ်ႇလႅၵ်ႈလၢႆႈတေႃႇသုတ်း (Static)၊ ႁဵတ်းၵၢၼ်ဝႆး လႄႈ ၵွင်ႉ Table တင်းပိူၼ်ႈငၢႆႈ။
* **Disadvantage:** ၶေႃႈမုၼ်းၼႆႉ ဢမ်ႇမီးတီႈပွင်ႇသင် တႃႇၵူၼ်းၸႂ်ႉ App ၶႃႈ။

### 4. Auto-Increment (ၵၢၼ်ၼပ်ႉမၢႆႁင်းၶေႃ)
ၼႂ်း MySQL ႁဝ်းၵႆႉၸႂ်ႉ **`AUTO_INCREMENT`** ၸွမ်း Surrogate Key ၶႃႈ။
* **How it works:** မိူဝ်ႈႁဝ်းသႂ်ႇၶေႃႈမုၼ်း (INSERT) မႂ်ႇ... Database တေၼပ်ႉမၢႆပၼ်ႁင်းမၼ်း 1, 2, 3... ၵႂႃႇၶႃႈ။
* ႁဝ်းဢမ်ႇလူဝ်ႇပႂ်ႉတူၺ်းဝႃႈ "မၢႆလိုၼ်းသုတ်းပဵၼ်မၢႆလႂ်" ၼႆၶႃႈ။



### 5. ပၵ်းၵၢၼ်တွင်း တွၼ်ႈတႃႇ Key Strategies
* **Simple is Best:** ၵမ်ႈၼမ် တႃႇတႅမ်ႈ App ၼၼ်ႉ ႁဝ်းၵႆႉၸႂ်ႉ **Surrogate Key** (ID) ဢၼ်ပဵၼ် **AUTO_INCREMENT** ၶႃႈ။
* **UUID:** သင်ပဵၼ် System လူင် ဢၼ်ၵွင်ႉၵၢႆႇၵၼ်လၢႆလၢႆ Server... ၶဝ်ၵႆႉၸႂ်ႉ UUID (လိၵ်ႈလႄႈတူဝ်ၼပ်ႉယၢဝ်းယၢဝ်း) တႅၼ်း Auto-Increment ၶႃႈ။

---

### ၶေႃႈၵႂၢမ်း Technical ဢၼ်လူဝ်ႇမၢႆတွင်း

* **Natural Key:** ၶေႃႈမုၼ်းဢၼ်မီးယူႇယဝ်ႉ ၼႂ်းတူဝ် Entity။
* **Surrogate Key:** ၶေႃႈမုၼ်း (ID) ဢၼ်သၢင်ႈမႂ်ႇ တႃႇၸႂ်ႉၼႂ်း Database ၵူၺ်း။
* **Auto-Increment:** ၵၢၼ်ပၼ် Database ၼပ်ႉမၢႆ ID ပၼ်ႁင်းၵူၺ်း။
* **Composite Key:** ၵၢၼ်ဢဝ် Column 2 ဢၼ် မႃးႁူမ်ႈၵၼ် ႁဵတ်းပဵၼ် Primary Key ၼိုင်ႈဢၼ်။
